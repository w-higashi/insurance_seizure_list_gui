# 生命保険差押予定一覧 作成ツール（insurance_seizure_list）GUI版 機能要件書

## 1. ツールの目的

pipitLINQ経由で取得した生命保険照会結果（.xlsm）から、差押対象の保険契約を職員が確認し、WinActorで読み込み可能なCSV（生命保険差押予定一覧.csv）を生成する。同時に、差押起案の印刷・証跡用として照会結果ファイルを文書番号付きで保存する。現在職員が手作業で行っている契約確認・差押文言作成・一覧転記・ファイル整理を半自動化し、ミスの削減と処理速度の向上を実現する。

本ツールはGUI版（.cs/.exe）として直接開発されており、コンソール版（.ps1）は存在しない。預金版 deposit_seizure_list のデザインシステム・アーキテクチャ・Excel COM パターンをベースに、生命保険照会結果固有のデータ構造に対応した新規ツールである。

## 3. 起動方式

### 3.1 直接起動
- insurance_seizure_list.exe をダブルクリックで起動する。
- 初期画面（ドラッグ＆ドロップゾーン）が表示される。

### 3.2 ドラッグ＆ドロップ起動
- .xlsm ファイルを exe またはショートカットにD&Dして起動する。
- 複数ファイルを一度に処理可能（順次処理）。

### 3.3 コマンドライン引数起動
- file_search.exe の「差押リストに追加」から呼び出される。
- 引数で渡されたファイルパスを処理キューに追加する。
- `--log=` プレフィックス付きの引数は差押登録ログの出力先パスとして分離する（ファイルパスには追加しない）。
- `isFromFileSearch = true` となり、全ファイル処理完了後にアプリを自動クローズする。

## 5. 基本情報セクション

### 5.1 レイアウト
- 外側Grid: `* | 16px | *` の3列（左右均等分割）。
- 左半分内側Grid: `120px | 16px | *`（宛名番号 + 氏名）。
- 右半分内側Grid: `2* | 16px | *`（保険会社 + 処分担当、保険会社に多めの幅）。
- 住民票住所は左半分の全幅、届出住所は右半分の全幅を使用する。
- 執行日 + カレンダーPopup は左半分、チェックボックスは右半分に配置する。

### 5.2 読取専用フィールド
- 宛名番号（10桁0埋め、Consolas フォント、グレー背景）
- 保険会社名（グレー背景）

### 5.3 編集可能フィールド
- 氏名、処分担当（✎マーク付きラベル）
- 住民票住所、届出住所（✎マーク付きラベル）
- Excel から読み取った値が初期表示されるが、ユーザーが TextBox 上で直接書き換えることができる。「一覧に追加」押下時に TextBox の現在値が CSV に出力されるため、読取値の手動補正が可能。

### 5.4 執行日入力
- テキスト入力（Consolas フォント）+ カレンダーPopupボタン。
- 7桁和暦（例: 5080615）または西暦（例: 2026/06/15）を受け付ける。
- LostFocus 時に自動パース・バリデーションを実行し、不正値は赤枠で警告する。

### 5.5 カレンダーPopup
- Popup + Calendar コントロール（SelectionMode: SingleDate）。
- 日付選択時に自動で Popup を閉じ、入力欄に反映する。
- Popup 展開時に DisplayMode を Month にリセットし、前回の年/月選択ビューが残らないようにする。
- SelectedDates をクリアしてから開くことで、同じ日付の再クリックでも SelectedDatesChanged が発火する。

### 5.6 届出住所
- 照会側住所（addressCell）が住民票住所、金融機関側住所（respAddressCell）が届出住所に対応する。
- チェックボックス「届出住所を差押通知書に出力する」をONにすると、CSV出力時に「（届出：{届出住所}）」形式で出力する。
- 50文字制限のバリデーションを実行し、超過時は赤色テキストで文字数を表示する。

## 8. 非同期処理

### 8.1 BackgroundWorker パターン
- Excel COM 操作はすべて BackgroundWorker で非同期実行する。
- UI スレッドのブロックを防ぎ、処理中はローディングオーバーレイ（スピナー）を表示する。

### 8.2 非同期処理の発生箇所
| 処理 | トリガー | BackgroundWorker の内容 |
|------|---------|----------------------|
| ファイル読込 | D&D / ファイル選択 / 次のファイルへ | ReadExcelFile（COM でファイルオープン→シートデータ読取） |
| シート切替 | ComboBox 選択変更 | ReadSheetData（選択シートのデータ再読取） |
| 一覧に追加 | ボタン押下 | ProcessAdd（文書番号採番→差押文言生成→CSV追記→印刷ファイル保存） |

### 8.3 冗長読込の抑止
- PopulateForm 内で sheetCombo.SelectedIndex を設定する際、suppressSheetChange フラグで SelectionChanged を抑止する。
- ReadExcelFile で初回シートのデータを既に取得済みのため、二重の COM アクセスを回避する。

### 8.4 Excel COM 一括取得（Value2 バルクリード）
- シート全体を `ws.Range["A1", "AN75"].Value2` で1回の COM 呼出しで一括取得する。
- 75行 × 40列 = 約3,000セルの 2D配列を取得し、`BulkCellByAddress` でセルアドレスから直接値を参照する。
- 預金版の複数回 COM 呼出し（表紙列スキャン + 口座テーブル範囲取得）に比べ、大幅に高速化されている。
- 給付金額（最大4件）と特約（最大10件）も同一の一括取得データからループで読み取る。

## 13. file_search.exe との連携

### 13.1 呼び出し方式
- file_search.cs の AddToSeizureList メソッドが insurance_seizure_list.exe を子プロセスとして起動する。
- 引数にファイルパスを渡す（複数ファイルの場合はスペース区切り）。
- `--log=` 引数で差押登録ログファイルのパスを渡す。
- UseShellExecute = false、EnableRaisingEvents = true で起動し、Process.Exited でログ再読み込みを行う。

### 13.2 設定
- file_search_config.json に生命保険版の exe パスを設定する（設定項目名は要検討）。

### 13.3 差押登録ログ
- 登録完了時（`result["status"] == "ok"`）に、`--log=` で渡されたパスに JSON ログを書き出す。
- `--log=` 引数がない場合（単体起動時）はログ書き出しをスキップする。
- 書き出し項目: 宛名番号、氏名、保険会社名、執行日（yyyy-MM-dd）、文書番号、登録日時（ISO）。
- 排他ロック + リトライ3回。全体を try/catch で囲み、失敗してもメイン処理に影響しない。

## 14. データ加工処理

### 14.1 金融機関名マッピング
- institution_name_mapping.json により保険会社名を補正する。
- 比較時に半角→全角正規化（ToFullWidth）を行い、表記ゆれを吸収する。
- ファイルが存在しない場合は補正なし。

### 14.2 住所加工
- 郵便番号（〒xxx-xxxx）と直後のスペース・全角スペースを除去する。
- 空白（半角・全角）を除去する。

### 14.3 日付パース
- Excel セルの値を DateTime に変換する（シリアル値・文字列の両方に対応）。
- 「2013年02月11日」形式の文字列パースにも対応する。

### 14.4 金額パース
- Excel セルの値を double に変換する（数値・カンマ付き文字列・「円」付き文字列に対応）。

## 15. 文書番号の採番

### 15.1 排他ロック
- insurance_document_number_counter.json を FileStream 排他ロック（FileShare.None）で開き、「読み取り→採番→書き戻し」をアトミックに実行する。
- 預金版（document_number_counter.json）とは独立したカウンターを使用する。

### 15.2 リトライ
- ロック取得に失敗した場合は最大5回、500ms間隔でリトライする。

### 15.3 CSV書き込み失敗時のロールバック
- CSV書き込みが全リトライ失敗した場合、消費済みの文書番号カウンターを1つ戻す。

## 16. 差押文言の生成

### 16.1 差押文言の構成
- 6つのフィールド（差押文言1〜6）に分割して出力する。
- 預金版（3フィールド）と異なり、保険契約の特定情報を個別列で出力する。

### 16.2 差押文言1（宣言文）
```
　上記滞納者が、債務者である{保険会社名（全角）}に対して有する下記生命保険契約に基づく
一切の支払請求権。なお、保険事故発生又は契約期間の終了しないうちに解約されたときは
解約返戻金全額の支払請求権。
```

### 16.3 差押文言2〜6（契約特定情報）
| フィールド | 内容 | 変換 |
|-----------|------|------|
| 差押文言2 | 保険の種類（契約種類） | 全角変換（ToFullWidth） |
| 差押文言3 | 証券番号 | 全角変換 |
| 差押文言4 | 保険契約年月日 | 和暦全角（例: 平成２５年２月１１日）、ゼロ埋めなし |
| 差押文言5 | 契約者 | そのまま |
| 差押文言6 | 被保険者 | そのまま |

### 16.4 和暦全角変換
- era_mapping.json を使用して西暦から元号を逆引きする。
- 数字は全角に変換し、月・日のゼロ埋めは行わない。
- 例: DateTime(2013,2,11) → 「平成２５年２月１１日」

## 17. CSV出力

### 17.1 出力ファイル
- ファイル名：「生命保険差押予定一覧.csv」（config の outputFolder 配下）。

### 17.2 追記方式

### 18.1 保存先
- printFolder（config で設定）配下に「文書番号.xlsm」として保存する。

### 18.2 処理内容

## 21. 預金版（deposit_seizure_list）との主な差異

| 項目 | 預金版 | 生命保険版 |
|------|--------|-----------|
| カラーテーマ | ティール #00897B | アンバー #E69500 |
| ウィンドウサイズ | 1000×700 | 1000×700 |
| データ構造 | 表紙ブロック + 口座テーブル（行ループ） | 1シート=1契約（固定セル） |
| Excel 読取り | 複数 COM 呼出し（列スキャン + テーブル範囲） | 1回の COM 呼出し（A1:AN75 一括取得） |
| 口座選択 | ListView + GridView（6列テーブル） | なし（シート選択 = 契約選択） |
| 画面構成 | 基本情報 + 口座テーブル | 基本情報 + 契約情報 + 金額情報（2カラム） |
| 差押文言 | 3フィールド（預金払戻請求権） | 6フィールド（保険契約特定情報を個別列） |
| CSV列数 | 20列 | 21列 |
| 差押実績照合 | あり（宛名番号+口座番号でCSV照合、★マーク） | なし（保険差押=解約のため再差押は発生しない） |
| 印刷ファイル保存 | シート削除 + 表紙ブロック削除 + 明細ページ削除 | シート削除のみ |
| 文書番号カウンター | document_number_counter.json | insurance_document_number_counter.json（独立） |
| マッピングファイル | 5種（振込手数料・ゆうちょ・口座種別・金融機関名・支店名） | 1種（金融機関名のみ） |
| 警告表示 | なし | 保険契約無 / 差押権利者有をインライン警告 |
| 差引見込額 | なし | 解約返戻金+配当金−貸付金−未払い保険料−未払い利息+前払い保険料 |
| コンソール版 | あり（.ps1） | なし（GUI版のみ） |
